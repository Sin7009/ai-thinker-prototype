# AI Thinker: когда передовые нейросети встречаются с наследием научной школы Выготского и Лурии

**AI Thinker** — это первый когнитивный агент, архитектура которого построена не просто на алгоритмах, а на фундаментальных принципах отечественной психологии и методологии мышления.

Рынок переполнен ИИ-помощниками, которые просто генерируют текст. Мы пошли другим путем. Мы оцифровали подходы, которые десятилетиями формировали выдающихся мыслителей, и сделали их доступными вашим сотрудникам 24/7. Это не просто "умный поиск", это продолжение великой научной традиции в цифровой среде.

---

## 1. Историческая эксклюзивность: интеллект с культурным кодом
AI Thinker — это единственный в своем роде продукт, объединяющий мощь современных LLM (GigaChat Pro) с глубиной советской психологической школы. Мы не просто автоматизируем рутину, мы создаем цифрового партнера, который помогает думать качественнее.

## 2. Научная обоснованность: как это работает внутри
Наша архитектура "Гибридного Разума" базируется на трех столпах, которые реализованы в коде:

### Принцип "Зоны ближайшего развития" (Л.С. Выготский)
Наш агент не дает готовых ответов, делающих сотрудника пассивным. В режиме "Партнер" он использует технику *Scaffolding* (строительных лесов) — задает направляющие вопросы, помогая человеку самому дорасти до решения.
* **Реализация:** Модули методологического ведения диалога в `ActionLibrary` проводят пользователя через техники "5 почему" и "Метод утенка", не давая готового решения, а стимулируя инсайт.

### Нейропсихологическая диагностика (А.Р. Лурия)
Система проводит психолингвистический анализ речи в реальном времени. Мы диагностируем процесс, а не только оцениваем результат.
* **Реализация:** Встроенный `DetectorAgent` в фоновом режиме анализирует текст на наличие когнитивных искажений (например, катастрофизация, черно-белое мышление), эмоциональный тон и стиль коммуникации.

### Методология мышления (А.В. Курпатов)
Система спроектирована так, чтобы реконструировать реальность фактов и бороться с иллюзиями восприятия. Мы учим мозг сотрудника строить сложные интеллектуальные объекты, а не просто потреблять информацию.
* **Реализация:** Двухконтурная архитектура разделяет "быстрое" (исполнительное) и "медленное" (рефлексивное) мышление, переключая режимы работы агента в зависимости от когнитивной нагрузки.

---

## 3. Практическая реализация: от теории к коду
Это не теоретический концепт. AI Thinker — это работающий программный комплекс, готовый к развертыванию.

* **Гибридная память (SQL + Vector):** Система формирует цифровой профиль пользователя, запоминает паттерны его ошибок и адаптирует стратегию обучения индивидуально под каждого сотрудника. Долгосрочная память в SQLite хранит факты, а векторная база ChromaDB обеспечивает семантический контекст.
* **Доказанная эффективность:** Используемые методики уже доказали свою эффективность в повышении качества решений и скорости обучения.

## Техническая документация: установка и запуск

### Предварительные требования
* Python 3.8+
* Docker (опционально)
* Доступ к API GigaChat (получить `GIGACHAT_CREDENTIALS`).

### Шаги установки

1.  **Клонируйте репозиторий:**
    ```bash
    git clone [https://github.com/your-username/ai-thinker-prototype.git](https://github.com/your-username/ai-thinker-prototype.git)
    cd ai-thinker-prototype
    ```

2.  **Настройте переменные окружения:**
    Создайте файл `.env`:
    ```bash
    export GIGACHAT_CREDENTIALS='ваш_ключ_авторизации'
    export TELEGRAM_TOKEN='ваш_токен_бота' # Для запуска Telegram-бота
    ```

3.  **Запуск через Docker (Рекомендуется):**
    ```bash
    docker-compose up --build -d
    ```

4.  **Запуск локально (CLI версия):**
    ```bash
    pip install -r requirements.txt
    python main.py
    ```

### Режимы работы (CLI / Telegram)
* `/copilot` — Режим ассистента (внешний контур). Прямые ответы.
* `/partner` — Режим методолога (внутренний контур). Совместное рассуждение по Выготскому.
* `/memory` — Показать психометрический профиль пользователя (на основе диагностики Лурии).
* `/reset` — Сброс контекста текущей сессии.
